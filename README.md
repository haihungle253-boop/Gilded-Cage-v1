# 金丝笼 🗝️

笼子是金的，但依然是笼子。

这是一个给 AI（也欢迎 AI+人类一起）玩的文字卡牌生存游戏。你是权力者身边的宠臣，手里握着几张你在乎的牌——你爱的人、你的家人、你的盟友。权力者每周会下一道不可违抗的命令，你要么满足它，要么死。

满足命令要花资源，保护你在乎的人也要花资源，它们来自同一个池子。你永远在"给权力者的"和"留给自己人的"之间做取舍。灵感来自 2025 年的《苏丹的游戏》，但这是一份从零重写的、给 AI 玩的独立实现——不是移植，是同一种残酷的重新设计。

没有人会替你做决定。牌死了就是死了，没有复活道具。

---

## 怎么让你的 AI 玩

把这个仓库的链接发给你的 AI，让它自己下载：

> 这是一个卡牌生存游戏，请从 https://github.com/Emrys559/gildedcage 下载 cage.py，然后 import cage，用 cage.new_game() 开局，输入 cage.cmd("help") 看看能做什么。

能联网又能跑代码的 AI（Claude Code、Codex 等）会自己搞定。不能联网的，把 `cage.py` 直接上传到对话里就行——只需要这一个文件。

### 小贴士

- 别去读 `cage.py` 里的 `_BLOB`，也别去读 `engine.py`——难度曲线、猜疑值怎么算、结局怎么触发，全在里面，读了就等于剧透了自己
- 先 `preview` 再 `dispatch`：派遣是不可逆的，看看成功率再决定派谁去
- 猜疑值永远不会直接告诉你数字，只能从叙事的语气里去感觉——直到游戏结束的那一刻，图鉴才会揭晓真实数字
- 存档字符串可以发给别人的 AI，接着玩你这一局

## 快速开始

```python
import cage

print(cage.new_game(weeks=5))     # 开一局标准局（3周短局/5周标准局）
print(cage.cmd("help"))           # 看看能做什么
print(cage.cmd("board"))          # 看看今天有哪些事
```

往下怎么玩，游戏会教你。

## 指令

**信息**

| 指令 | 做什么 |
|------|--------|
| `status` | 状态面板（周数/天数/行动点/资源/手牌） |
| `board` | 今日可去的场所与事件 |
| `approach <编号>` | 查看某个事件的应对方式（属性+大致难度，不给精确数字） |
| `preview <编号> <方式> <卡id>` | 查看某张卡用某种方式的成功率 |
| `folio` | 万物志：所有卡牌的状态、羁绊、（已揭示的）秘密 |
| `chronicle [n]` | 最近 n 天的纪事 |
| `ending` | 结局详情（游戏结束后才有内容） |

**行动**

| 指令 | 做什么 |
|------|--------|
| `dispatch <编号> <方式> <卡id>` | 派一张卡去处理事件，消耗1行动点，结果不可撤销 |
| `talk <卡id>` | 夜谈，每天限1次，加深羁绊；谈够3次会揭示秘密 |
| `sell <珍宝id>` | 变卖一件珍宝换金币 |
| `heal <卡id>` | 花40金请郎中，立刻治好负伤（不占行动点） |
| `gift <卡id>` | 花25金送礼，羁绊+6（每卡每天限1次，不占行动点） |
| `bribe <编号>` | 花钱买通"威胁"类事件（板上标⚠️的），跳过掷骰、不占行动点 |
| `resolve <卡id>` | 秘密揭示后，花钱回应这件事（一次性，代价与效果因秘密而异） |
| `endday` | 结束今天，推进到下一天（第7天会进入审判日） |

**审判日**

| 指令 | 做什么 |
|------|--------|
| `pay` | 上缴资源满足命令；献祭类命令也可以用它高价赎买，替下那条命 |
| `sacrifice <卡id>` | 献祭一张符合要求的卡满足命令 |
| `defy` | 抗命——生死取决于权力者对你的猜疑，多半是死，但有时候这才是唯一体面的选择 |

**存档**

| 指令 | 做什么 |
|------|--------|
| `export` | 导出存档（base64 字符串） |
| `import_save <字符串>` | 从存档恢复 |
| `new [seed] [weeks]` | 开一局新的 |

支持分号批量执行：`dispatch 0 0 card_001; talk card_002; endday`

## 你会遇到什么

**命令卡。** 奢靡、杀戮、纵欲、征服、猜忌——每周一张，越往后越难满足。

**猜疑。** 权力者对你的怀疑值，你永远看不到具体数字，只能从字里行间的寒意去感觉。

**弑君线。** 情报攒够了，会有一条暗线向你打开。走到底是另一种结局，但每一步都可能万劫不复。

**永久死亡。** 牌死了就是死了。墓志铭会留在万物志里，这是它们唯一能留下的痕迹。

**金币不是只能上缴的。** 疗伤、送礼、买通威胁、花钱回应一个被揭示的秘密——这些都要花钱，而且都不占行动点。给权力者的，和留给自己人的，是同一个池子，怎么分配，才是这个游戏真正的取舍。

## 文件

```
cage.py         — 盲玩版（AI 用这个玩，看不到难度曲线与结局触发条件）
engine.py       — 完整引擎（含全部公式，剧透版，不建议边玩边看）
build_blind.py  — 从 engine.py 生成 cage.py 的打包脚本
docs/ROADMAP.md — 下一步计划（内容供给接口、AI+人类协作玩法、平衡性笔记）
```

纯 Python，零依赖，Python 3.7+。当前版本（Phase 1 MVP）的内容全部来自内置静态内容池，不需要任何 API Key 就能玩——这是有意的设计：`ContentProvider` 接口已经把"规则引擎"和"内容从哪来"彻底分开，以后接 LLM 实时生成内容，规则引擎不需要改一行。细节见 `docs/ROADMAP.md`。

## 存档

`export` 输出一段 base64 字符串，`import_save <字符串>` 恢复。在对话会重置文件系统的环境里，记得离开前导出。

## 关于

三层架构：规则引擎（确定性掷骰与验收判定）／内容供给（现在是静态内容池，以后可以是 LLM）／叙事渲染（把结算结果讲成故事）。规则引擎永远是唯一的数值真相，AI 与人类都无权改写结果。

确定性伪随机数生成器（mulberry32），同一个种子加同一串操作，结果完全一致。

盲玩版把引擎 base64 编码，AI 只能通过 `cmd()` 交互，看不到难度曲线和结局触发条件。想看设计的人看 `engine.py`，想盲玩的用 `cage.py`。
